# 事实检查规则

本文档定义博客文章的事实核查要求，确保文章内容准确可靠。

## 绝对禁止

以下信息**严禁编造**，必须查证或标注：

### 数据类

- [ ] Star 数
- [ ] 下载量/安装量
- [ ] 用户数量
- [ ] 性能数据（速度、内存占用等）
- [ ] 统计数据

### 事实类

- [ ] 外部项目版本号
- [ ] 发布时间/日期
- [ ] 第三方服务的可用性
- [ ] 他人的言论或评价
- [ ] 开源协议的准确性

### 状态类

- [ ] 把未完成功能写成已完成
- [ ] 把计划中的功能写成已实现
- [ ] 把实验性功能写成稳定功能

---

## 强制标注

以下情况**必须标注**，使用 `TODO` 或 `[待确认]` 或 `[不确定]`：

### 需要验证但暂无来源

```
- 估计用户数：约 1000 人 [待确认]
- 这个功能在 v1.2 加入 [不确定]
```

### 引用外部信息

```
- 引用自 GitHub README
- 数据来源：npm weekly downloads
```

### 时间相关

```
- 20XX 年 [待确认具体日期]
- 最近 [不精确的时间描述]
```

---

## 验证清单

发布前逐项检查：

### 必检项

- [ ] 所有数字是否与官方数据一致
- [ ] 所有链接是否有效
- [ ] 所有版本号是否正确
- [ ] 所有日期是否准确
- [ ] 代码示例是否能正常运行

### 建议检项

- [ ] 术语使用是否准确
- [ ] 截图是否反映最新版本
- [ ] 功能描述是否与代码一致

## 常见问题

### Q: 不知道确切数字怎么办？

A: 使用模糊表述或标注来源

```
# 不要这样写
项目已有 5000 用户

# 可以这样写
项目已有数千用户 [数据来源：作者估计]

# 或者这样写
用户数暂无准确统计 [TODO: 待补充]
```

### Q: 外部链接失效怎么办？

A: 移除链接或标注

```
# 如果链接失效
Steamcommunity 302 是一个 [已停止维护] 的工具。

# 如果不确定
可以访问 [官方页面] 查看最新信息。
```

### Q: 功能已完成但还没发布怎么办？

A: 区分「开发完成」和「已发布」

```
# 不要这样写
已支持 X 功能

# 可以这样写
已在开发版中实现 X 功能 [待发布]
计划在下个版本发布 X 功能
```

---

## 惩罚措施

违反事实检查规则的文章：

- 退回修改
- 标注为「待核查版本」
- 不计入发布统计
