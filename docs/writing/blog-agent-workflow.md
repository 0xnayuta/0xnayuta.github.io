# Agent 辅助写博客流程

本文档定义使用 Agent（pi / Hermes）辅助写博客的标准流程。

## 概述

```
资料收集 → 资料包整理 → 大纲生成 → 草稿生成 → 事实检查 → 人工审阅 → 正式发布
```

所有草稿默认写入 `drafts/articles/`，未经明确要求不要直接写入 `src/data/blog/`。

---

## 详细流程

### 1. 资料收集

从项目仓库、README、代码、文档、Issue、Commit 历史等渠道收集写作素材。

**输出**：原始资料集合

### 2. 资料包整理

按照 `source-package-template.md` 的格式整理项目写作资料包。

**存放位置**：`drafts/sources/<项目名>/source.md`

**输出**：结构化的项目资料包

### 3. 文章大纲生成

根据资料包和 `article-template.md` 生成文章大纲。

**包含内容**：

- 标题建议
- 各章节要点
- 需要强调的技术细节
- 可能的配图建议

**输出**：文章大纲（供人工确认后再继续）

### 4. 草稿生成

根据确认后的大纲，使用 `article-template.md` 生成完整草稿。

**存放位置**：`drafts/articles/<日期>-<slug>.md`

**输出**：博客文章草稿

### 5. 事实检查

按照 `fact-check-rules.md` 对草稿进行事实核查：

- [ ] 验证所有数据、版本号、日期
- [ ] 标注不确定信息
- [ ] 标记需要来源引用的内容
- [ ] 确认未完成功能已标注

**输出**：事实核查报告或更新后的草稿

### 6. 人工审阅

人工检查草稿的：

- [ ] 内容准确性
- [ ] 表达流畅度
- [ ] 个人风格一致性
- [ ] 是否有遗漏的重要信息
- [ ] 配图和链接是否合适

**输出**：审阅意见

### 7. 移动到正式目录

根据审阅意见修改后，将文章移动到正式目录：

```bash
mv drafts/articles/<文件名> src/data/blog/<文件名>
```

**更新 frontmatter**：

- `pubDatetime` 设置为正式发布日期
- `draft: false`

### 8. 构建验证

执行以下命令验证：

```bash
pnpm format && pnpm lint && pnpm build
```

确认构建成功后，文章正式发布。

---

## 流程状态标记

| 状态   | 含义                                 |
| ------ | ------------------------------------ |
| 待审阅 | 草稿完成，等待人工审阅               |
| 需修改 | 需要 Agent 或人工修改                |
| 待发布 | 审阅通过，等待正式发布               |
| 已发布 | 已移动到 `src/data/blog/` 并构建成功 |

---

## 注意事项

1. **不要跳过人工审阅**：即使时间紧迫，也要有人工审核环节
2. **不确定信息必须标注**：使用 `TODO` 或 `[待确认]` 标记
3. **保持个人风格**：参考 `article-style-guide.md`
4. **一个项目一篇文章**：避免一篇博客堆砌太多项目
5. **按阶段提交**：不要一次性生成完整文章，分阶段确认
